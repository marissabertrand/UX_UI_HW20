Executive Summary

A team of University of Minnesota UX/UI Bootcamp students redesigned the website of the non-profit, The Women's Prison Book Project. The non-profit aims to provide literature to imprisoned women. The website was redesigned to condense pages and create better user paths.

Team Size: 3

Tools Used: Figma

Timeline: 4 Weeks

Deliverables: User Interview Data, Sitemap, Style Guide, Desktop and Mobile Prototypes

Although I contributed end-to-end on the redesign, I focused upon the user research, ideation, and desktop prototype.

To begin user research, we created a qualitative user interview plan, asking users about their reading habits, participation in book drives, engagement in non-profit websites, engagement with the prison population, and their opinion of the Women's Prison Book Project website. Priorities for users include the non-profit credibility and the mission of the organization. Users felt the website did not present sufficient general information, did not provide a financial disclosure, and lacked a verifiable claims, such as testimonials and statistics.

From the user research, we created an affinity map and empathy map, determining users prioritized the site content, aesthetics, credibility, and volunteer experience. We created the user persona Paige Brown, an avid reader who has old books she wants to ethically rehome, but is unable to donate through the non-profit Women's Prison Book Project due to the lack of clear information on the site.

We delved deeper by creating user insights, problem statements, and hypotheses, determining that if prospective volunteers had updated and verifiable content, they would be more likely to engage with the prison population through a trustworthy agency.

We first created a sitemap, re-organized the site content to reduce page length, then number of pages, and re-categorize information more logically. We created a Founders page, helping users understand the organization. We also created a Donate page, consolidating donating funds and donating books into a single page. Following establishing the sitemap, we then created a low-fidelity prototype and performed user testing. From the user feedback, we created a high-fidelity prototype, adding a hero image to the home page, streamlining the use of social media on the website, and using cards with hover states to shorten the page length.

We performed guerrilla user testing on a low-fidelity prototype, to which users stated they felt the home page needed a call to action donate button, and wanted more information on volunteer opportunities. From this data we were able to create the high-fidelity protoype.

The non-profit website redesign of the non-profit was successful, allowing users to better engage with the organization, and ultimately, the often forgotten prison population. To improve the redesign, we would have interviewed the non-profit stakeholders, who we were unable to contact, and we would also visit their volunteer events to have a better understanding of the processes and how a website could best serve the organization.
